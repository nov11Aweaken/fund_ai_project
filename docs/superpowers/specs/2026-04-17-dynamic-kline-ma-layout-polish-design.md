# 动态 K 线图 MA 选项布局优化设计

## 背景

当前外部动态图页面已经支持预设 MA 多选与即时重绘，但用户反馈两个实际问题：

1. 页面中只能直观看到 `MA5/MA10/MA20/MA250`，其余候选周期没有正常暴露出来。
2. 当前 MA 选项放在右上角工具区域内，位置和选中态都不够合理，影响识别与操作效率。

从当前实现看，固定候选周期 `5/10/20/30/60/120/250` 已经写入 HTML，但布局结构把 MA 选项挤在标题工具栏右侧，导致后半段候选在实际页面中不易展示或直接被挤出主要可视区域。

## 目标

1. 让 `5/10/20/30/60/120/250` 七个候选 MA 在页面中稳定可见。
2. 将 MA 选项从右上角工具栏调整到标题下方的独立整行区域。
3. 优化选中态，让用户能快速识别当前启用的 MA。
4. 保留当前即时重绘能力和缩放视窗保持能力。
5. 窗口宽度不足时，MA 选项自动换行到第二行。

## 非目标

1. 不新增自定义 MA 数字输入。
2. 不引入下拉面板、弹出层、折叠面板等新交互结构。
3. 不改动基金主界面布局。
4. 不引入持久化偏好，重新打开页面仍恢复默认 `5/10/20/250`。

## 方案比较

### 方案 A：标题下方独立胶囊行（最终选择）

- 标题与提示文案保留在首行。
- MA 选项独立成第二行，使用胶囊样式横向排列。
- 宽度不足时自动换行。

**优点**

- 七个候选周期都能稳定暴露。
- 视觉层级清晰，标题、提示、操作区各自独立。
- 最符合当前用户诉求，改动集中在 HTML 结构与样式层。

**缺点**

- 顶部区域整体高度会比当前版本略高。

### 方案 B：保留右上角并压缩工具区

- 继续放在右上角，只调整间距、字号和换行。

**优点**

- 改动最小。

**缺点**

- 在窄窗口或标题较长时仍容易拥挤。
- MA 选项和提示文案会继续争夺同一块视觉空间。

### 方案 C：图表上方悬浮操作条

- 在图表卡片内部顶部增加浮层式 MA 工具条。

**优点**

- 操作区离图表更近。

**缺点**

- 会与图表主体形成视觉竞争。
- 缩放、提示、导出等图表交互层级更复杂。

**最终选择：方案 A。**

## 交互设计

页面顶部调整为两层：

1. **标题行**
   - 左侧：标题
   - 右侧：简短提示文本

2. **MA 操作行**
   - 固定顺序：`MA5 / MA10 / MA20 / MA30 / MA60 / MA120 / MA250`
   - 默认选中：`MA5 / MA10 / MA20 / MA250`
   - 选中态：强调色实心胶囊 + 勾选标记
   - 未选中态：弱化背景或弱描边，保持可点击但不过度抢眼
   - 窄窗口下自动换行，不使用横向滚动

交互规则：

1. 点击任意 MA 胶囊立即重绘图表。
2. 单位净值主线始终显示。
3. 全部 MA 取消时，图表只保留单位净值主线。
4. 切换 MA 时保留用户当前缩放视窗。
5. 页面重新打开后恢复默认选中状态，不记忆上次勾选。

## 实现设计

### HTML 结构

在 `build_dynamic_chart_document()` 中：

1. 将现有顶部工具区拆成“标题行 + MA 操作行”两层结构。
2. MA 胶囊区不再嵌在右上角提示区内部。
3. 保持现有图表卡片与本地 `echarts.min.js` 引用方式不变。
4. 为测试和脚本交互保留稳定 DOM 标识：
   - MA 容器包含独立类名，例如 `ma-controls-row`
   - 每个候选胶囊保留 `data-ma-day="<周期>"` 标识
   - 选中态保留稳定类名，例如 `is-selected`
   - 勾选标记保留独立节点或类名，例如 `ma-check`

### 样式

1. 新增更明确的 MA 胶囊样式：
   - 默认态：浅背景/弱描边
   - 选中态：强调色实心
   - 勾选标记：显示在胶囊内部，强化当前状态
2. MA 容器使用可换行布局，确保七个候选都能露出。
3. 响应式验收口径：
   - 页面宽度在 `>= 480px` 时，不出现 MA 区横向滚动
   - 页面宽度不足 `480px` 时，允许自然换行到第二行
   - 不通过压缩到不可点击的小尺寸来强行维持单行
3. 继续保持现有页面卡片风格、中文字体与间距语义，不做整页重构。

### 数据与重绘逻辑

1. 保持现有 `build_dynamic_chart_data()`、`build_dynamic_chart_series()`、`build_dynamic_chart_option()` 数据契约不变。
2. 保持现有页面内 JavaScript 即时重绘逻辑不变。
3. 保持 `mergeCurrentZoom()` 机制，避免勾选变更后丢失当前缩放范围。
4. `get_chart_html()` 与 `write_dynamic_chart_html()` 继续统一走结构化图表数据链路。
5. 保留原生 `input[type="checkbox"]` 交互能力，使 MA 选项天然支持 TAB 聚焦与空格切换；本次不额外引入复杂无障碍增强脚本。

## 错误处理

1. 不改变当前动态图数据准备失败时的中文异常语义。
2. 不改变本地 ECharts 资源缺失时的错误提示。
3. 不因为 MA 全部取消而产生空图或报错。

## 测试设计

补充或更新回归测试，重点锁定：

1. HTML 中包含七个固定 MA 候选。
2. 新布局中标题区与 MA 独立操作区同时存在。
3. 默认选中仍为 `5/10/20/250`。
4. 新选中态所需的稳定类名或标记已经输出到 HTML，例如 `is-selected` / `ma-check`。
5. `get_chart_html()` 与 `write_dynamic_chart_html()` 仍然走结构化数据契约。
6. 不回退本地 `echarts.min.js` 引用。
7. MA 选项仍通过原生 checkbox 保持键盘可操作。
