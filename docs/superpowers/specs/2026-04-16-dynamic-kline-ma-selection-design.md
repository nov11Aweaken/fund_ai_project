# 动态 K 线图预设 MA 多选设计

## 背景

当前动态 K 线图已经修复为使用本地 `echarts.min.js` 资源，并在外部 HTML 页面中展示基金净值主线与若干固定 MA 线。现在需要在**外部动态图页面内部**增加预设 MA 多选能力，方便用户按需查看不同均线组合。

## 目标

1. 在外部动态图页面顶部提供预设 MA 多选控件。
2. 候选周期固定为 `5/10/20/30/60/120/250`。
3. 默认选中 `5/10/20/250`。
4. 用户切换勾选后立即重绘当前图表。
5. 每次重新打开动态图页面时恢复默认选中状态。

## 非目标

1. 不支持任意自定义 MA 数字输入。
2. 不做按基金保存或全局保存用户偏好。
3. 不修改基金详情页主界面布局，不在主界面新增额外交互控件。

## 方案选择

### 方案 A：页面内原生复选项 + 即时重绘

- 在外部 HTML 页面顶部渲染一行复选项。
- Python 端一次性提供候选 MA 数据与默认选中列表。
- 页面内通过原生 JavaScript 控制 ECharts `setOption()` 重绘。

**优点**
- 改动集中，最贴合现有“Python 生成本地 HTML + ECharts 渲染”的结构。
- 不需要重新请求 Python 端，不需要二次生成 HTML。
- 测试边界清晰，适合先写回归测试再实现。

**缺点**
- 需要在 HTML 文档中增加一段前端状态管理逻辑。

### 方案 B：页面内下拉多选面板

- 用一个按钮展开/收起多选面板，再控制 MA 线显示。

**优点**
- 页面更紧凑。

**缺点**
- 前端展开态与交互代码更多，对当前需求属于过度设计。

### 方案 C：按钮组开关

- 为每个周期提供按钮样式开关。

**优点**
- 外观更规整。

**缺点**
- 选中态样式和多选逻辑更重，收益不明显。

**最终选择：方案 A。**

## 交互设计

动态图页面由两部分组成：

1. 顶部工具栏  
   - 标题
   - 简短提示文本
   - `MA` 复选项组：`5/10/20/30/60/120/250`

2. 图表区域  
   - 单位净值主线始终显示
   - 仅显示当前选中的 MA 线

交互规则：

1. 首次打开页面时默认勾选 `5/10/20/250`。
2. 勾选变化后立即刷新图表。
3. 关闭页面后不保留勾选状态。
4. 不允许隐藏净值主线；MA 全部取消时，图表仍保留净值主线。

## 实现设计

### Python 端

保留 `write_dynamic_chart_html()` 作为外部动态图页面入口，并在现有链路上拆分为两层：

1. **数据计算层**
   - 基于历史净值数据统一计算候选 MA 序列。
   - 输出结构中包含：
     - 日期轴
     - 净值主线
     - 各候选 MA 序列
     - 默认选中列表
     - 候选周期列表

2. **HTML 组装层**
   - 负责把图表数据、默认选中状态、候选周期写入 HTML。
   - 页面引用本地 `echarts.min.js`，继续保持离线可打开。

### 前端页面

页面内增加一段原生 JavaScript：

1. 使用 Python 提供的数据初始化 ECharts。
2. 根据默认选中列表生成初始 `series`。
3. 动态渲染复选项。
4. 监听复选项变化并重新构建 `series` 后调用 `chart.setOption()`。
5. 仅更新图表配置，不重新加载页面，不重新请求本地文件。

## 错误处理

1. 历史数据为空或解析失败时，继续沿用当前动态图生成失败提示。
2. 如果候选 MA 数据构造异常，显式抛出 `ValueError`，由现有调用链向用户显示中文错误提示。
3. 页面内即使取消全部 MA，也应保持净值主线可渲染，避免出现空图。

## 测试设计

先补失败测试，再实现功能。重点覆盖：

1. 数据结构中包含固定候选周期和默认选中列表。
2. 默认配置仅生成 `MA5/MA10/MA20/MA250`。
3. 自定义选中列表时，只输出对应 MA 线。
4. 生成的 HTML 中包含候选周期与默认选中配置。
5. 现有“打开动态图失败提示”逻辑不回退。
