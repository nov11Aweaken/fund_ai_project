# 动态 K 线图预设 MA 多选设计

## 背景

当前动态 K 线图已经修复为使用本地 `echarts.min.js` 资源，并在外部 HTML 页面中展示基金净值主线与若干固定 MA 线。现在需要在**外部动态图页面内部**增加预设 MA 多选能力，方便用户按需查看不同均线组合。

## 目标

1. 在外部动态图页面顶部提供预设 MA 多选控件。
2. 候选周期固定为 `5/10/20/30/60/120/250`。
3. 默认选中 `5/10/20/250`。
4. 用户切换勾选后立即重绘当前图表。
5. 每次重新打开动态图页面时恢复默认选中状态。

## 非目标

1. 不支持任意自定义 MA 数字输入。
2. 不做按基金保存或全局保存用户偏好。
3. 不修改基金详情页主界面布局，不在主界面新增额外交互控件。

## 方案选择

### 方案 A：页面内原生复选项 + 即时重绘

- 在外部 HTML 页面顶部渲染一行复选项。
- Python 端一次性提供候选 MA 数据与默认选中列表。
- 页面内通过原生 JavaScript 控制 ECharts `setOption()` 重绘。

**优点**
- 改动集中，最贴合现有“Python 生成本地 HTML + ECharts 渲染”的结构。
- 不需要重新请求 Python 端，不需要二次生成 HTML。
- 测试边界清晰，适合先写回归测试再实现。

**缺点**
- 需要在 HTML 文档中增加一段前端状态管理逻辑。

### 方案 B：页面内下拉多选面板

- 用一个按钮展开/收起多选面板，再控制 MA 线显示。

**优点**
- 页面更紧凑。

**缺点**
- 前端展开态与交互代码更多，对当前需求属于过度设计。

### 方案 C：按钮组开关

- 为每个周期提供按钮样式开关。

**优点**
- 外观更规整。

**缺点**
- 选中态样式和多选逻辑更重，收益不明显。

**最终选择：方案 A。**

## 交互设计

动态图页面由两部分组成：

1. 顶部工具栏  
   - 标题
   - 简短提示文本
   - `MA` 复选项组：`5/10/20/30/60/120/250`

2. 图表区域  
   - 单位净值主线始终显示
   - 仅显示当前选中的 MA 线

交互规则：

1. 首次打开页面时默认勾选 `5/10/20/250`。
2. 勾选变化后立即刷新图表。
3. 关闭页面后不保留勾选状态。
4. 不允许隐藏净值主线；MA 全部取消时，图表仍保留净值主线。

## 实现设计

### Python 端

保留 `write_dynamic_chart_html()` 作为外部动态图页面入口，但把现有“直接输出完整 `option_json`”的职责拆清楚：

1. **数据计算层：新增结构化图表数据函数**
   - 基于历史净值数据统一计算候选 MA 序列。
   - 输出一个稳定字典，至少包含：
     - `title`
     - `dates`
     - `nav_values`
     - `ma_series`：`{5: [...], 10: [...], 20: [...], 30: [...], 60: [...], 120: [...], 250: [...]}`
     - `ma_candidates`：`[5, 10, 20, 30, 60, 120, 250]`
     - `default_ma_days`：`[5, 10, 20, 250]`

2. **图表组装层：只负责把结构化数据写入 HTML**
   - `build_dynamic_chart_document()` 不再只吃最终 `option_json`，而是接收上面的结构化数据。
   - 页面内 JavaScript 根据当前勾选状态构造 `series`，并调用 ECharts `setOption()`。
   - 这样职责明确：Python 负责准备数据，浏览器负责切换显示，不新增新的 Python 交互接口。

3. **兼容边界**
   - `write_dynamic_chart_html()` 仍然只接收 `tgt`，不新增“外部传入选中 MA 列表”的参数。
   - 每次打开页面都使用文档约定的默认选中列表，不把页面内临时勾选回写到 Python 端。

### 前端页面

页面内增加一段原生 JavaScript：

1. 使用 Python 提供的数据初始化 ECharts。
2. 根据默认选中列表生成初始 `series`。
3. 动态渲染复选项。
4. 监听复选项变化并重新构建 `series` 后调用 `chart.setOption()`。
5. 仅更新图表配置，不重新加载页面，不重新请求本地文件。

## 错误处理

1. 历史数据为空或解析失败时，继续沿用当前动态图生成失败提示。
2. 如果候选 MA 数据构造异常，显式抛出 `ValueError`，由现有调用链向用户显示中文错误提示。
3. 页面内即使取消全部 MA，也应保持净值主线可渲染，避免出现空图。

## 测试设计

先补失败测试，再实现功能。重点覆盖：

1. 结构化图表数据中包含固定候选周期和默认选中列表。
2. 生成的 HTML 中包含候选周期、默认选中配置和页面内重绘所需的序列数据。
3. 默认首次渲染只显示 `MA5/MA10/MA20/MA250`。
4. 页面内切换勾选时，重建后的 `series` 只包含当前选中的 MA 线。
5. 页面内取消全部 MA 时，仍保留单位净值主线。
6. 关闭页面再重新打开时，恢复默认 `5/10/20/250`。
7. 现有“打开动态图失败提示”逻辑不回退。
